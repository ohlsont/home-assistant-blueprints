# Automatic Release on Main Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Automatically publish a GitHub release when a new synced Blockheat version lands on `main`.

**Architecture:** Add a small Python helper that validates the repo version state across both manifests and `pyproject.toml`, then wire a dedicated GitHub Actions workflow that reruns verification on `main` and publishes `vX.Y.Z` only after those jobs pass. Update docs and existing CI so the new helper is covered by tests and linted in PRs.

**Tech Stack:** GitHub Actions, Python 3.14, `uv`, `pytest`, `ruff`

---

### Task 1: Add version-validation tests

**Files:**
- Create: `tests/test_release_version_validation.py`
- Test: `tests/test_release_version_validation.py`

**Step 1: Write the failing tests**
- Cover matching versions, manifest mismatch, `pyproject.toml` mismatch, and invalid or missing version values.
- Import the script module from `scripts/validate_release_version.py`.

**Step 2: Run the targeted test file**
- Run: `uv run pytest tests/test_release_version_validation.py -q`
- Expected: FAIL because the helper script does not exist yet.

**Step 3: Write the minimal helper implementation**
- Create `scripts/validate_release_version.py`.
- Expose reusable functions for reading versions and validating that they are all equal.
- Print the resolved version on success and a clear error on failure.

**Step 4: Re-run the targeted test file**
- Run: `uv run pytest tests/test_release_version_validation.py -q`
- Expected: PASS.

### Task 2: Wire CI and release automation

**Files:**
- Modify: `.github/workflows/blockheat-mirror-sync.yml`
- Create: `.github/workflows/release.yml`
- Modify: `pyproject.toml`

**Step 1: Update existing CI coverage**
- Include `scripts` in ruff format and lint commands so the new helper is checked in PR CI.
- Sync `pyproject.toml` version with the manifest version.
- Update mirror-sync workflow to use `uv` consistently for the script execution path.

**Step 2: Add the release workflow**
- Trigger on pushes to `main`.
- Re-run mirror sync, tests, formatting/lint, and mypy.
- Resolve the release version via the helper.
- Fail if `vX.Y.Z` already exists.
- Publish a GitHub Release with autogenerated notes if verification passes.

**Step 3: Run focused local verification**
- Run: `uv run pytest tests/test_release_version_validation.py tests/test_mirror_sync.py -q`
- Run: `uv run ruff check scripts tests`
- Expected: PASS.
